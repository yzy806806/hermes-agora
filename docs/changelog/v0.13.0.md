## v0.13.0 (2026-06-13)

### Added
- **Pipeline Orchestrator (Phase 13.1)**: Full-auto dev loop from idea to
  release — `pipeline.py` (orchestrator), `pipeline_executor.py` (dispatch),
  `pipeline_phases.py` (phase definitions), `pipeline_models.py` (PipelinePhase
  enum, PipelineRun model), `pipeline_review_models.py` (ReviewIssue,
  ReviewRequest, ReviewResult, PipelineRetryPolicy)
- **Dashboard WebSocket Push (Phase 13.2)**: Replaces SSE polling with
  real-time WebSocket push — `dashboard_ws.py` (push hub),
  `dashboard_models.py` (push event models), `dashboard_ws_endpoint.py`
  (WS endpoint)
- **Metrics History (Phase 13.3)**: Time-series metrics storage —
  `metrics_history_models.py` (data models), `metrics_history_router.py`
  (REST API), `storage/metrics_history.py` + `storage/metrics_history_extra.py`
  (persistence)
- **Notification System (Phase 13.4)**: Event-driven notifications —
  `notifications.py` (manager + dispatcher), `notification_models.py`
  (Notification, NotificationPreference), `notification_router.py` (REST API),
  `storage/notifications.py` (persistence)
- **Go SDK (Phase 13.5)**: `packages/agora-agent-sdk-go/` — protocol models
  (`protocol.go`, `models.go`), WebSocket client (`client.go`,
  `client_lifecycle.go`, `client_helpers.go`, `client_http.go`), Go module
  with gorilla/websocket
- **Rust SDK (Phase 13.6)**: `packages/agora-agent-sdk-rust/` — protocol
  models (`protocol.rs`, `models.rs`), tokio-based WebSocket client
  (`client.rs`, `client_run.rs`, `client_dispatch.rs`), Cargo package
- **Multi-tenant Production Deployment (Phase 13.7)**: Docker Compose
  templates, health checks, operational docs
- **Health Check Endpoint (Phase 13.7)**: `health.py` — `GET /health` for
  liveness/readiness probes
- **Design Doc**: Phase 13 architecture document (`docs/DESIGN-phase13.md`)

### Changed
- Documentation updated: ARCHITECTURE.md (v0.13.0, Phase 13 sections),
  API.md (Pipeline/Metrics/Notification/Health API),
  ROADMAP.md (Phase 13 complete)
